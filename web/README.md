# Django Web Frontend

Веб-интерфейс для спортивного магазина, работающий через API.

## Описание

Этот проект представляет собой веб-интерфейс (frontend), который взаимодействует с основным API, развернутым на `http://127.0.0.1:8000/`. Сам веб-сайт работает на порту 8001.

## Установка

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Выполните миграции для сессий Django:
```bash
python manage.py migrate
```

3. Создайте суперпользователя (опционально, для админ-панели):
```bash
python manage.py createsuperuser
```

## Запуск

### Windows:
```bash
runserver.bat
```

### Linux/Mac:
```bash
python manage.py runserver 8001
```

Сайт будет доступен по адресу: `http://127.0.0.1:8001`

## Настройки

Настройки API находятся в `website/settings.py`:
- `API_BASE_URL` - базовый URL API (по умолчанию: `http://127.0.0.1:8000/api/v1`)

Можно изменить через переменные окружения или файл `.env`:
```
API_BASE_URL=http://127.0.0.1:8000/api/v1
```

## Структура проекта

- `shop/` - основное приложение веб-интерфейса
  - `api_client.py` - клиент для работы с API
  - `views.py` - представления (работают через API)
  - `forms.py` - формы
  - `urls.py` - URL маршруты
- `templates/` - HTML шаблоны
- `static/` - статические файлы (CSS, JS, изображения)
- `website/` - настройки Django проекта

## Особенности

- Все данные получаются через API, нет прямого доступа к БД
- Аутентификация через JWT токены, хранящиеся в сессии
- Полная интеграция с основным API проекта
